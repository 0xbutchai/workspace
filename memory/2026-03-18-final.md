## Fast Build Path (Optimized)

Original plan was to render every frame at 1080p (slow). Optimized to:
1. Render 1 static composite frame (instant)
2. Render only 48 animation frames (2 sec of ball arc at 24fps)
3. Use FFmpeg to stitch: static → animation → static = 60s loop
4. Repeat loop 18x to match audio duration

**Result:** ~30 min total build time vs. estimated 3+ hours for full frame-by-frame render.

## Lessons Learned

1. **Static frames >> full render**: Compositing every frame is expensive. Precompute statics, animate only what moves.
2. **FFmpeg concat > re-encoding**: Use demuxer concat for video loops and audio, keep codecs in sync to avoid re-encode.
3. **MoviePy 2.x API broke**: `moviepy.editor` module doesn't exist; use `from moviepy import VideoClip`.
4. **Opus codec friction**: Suno outputs Opus (good quality-to-bitrate), but MP4/M4A containers are picky. Transcode to AAC for containers that enforce it.
5. **Simplify the target**: 6 best tracks + ~17m video > 12+ tracks + trying for full hour. Ship something good, iterate later.
