## Lofi Shaman Animation — COMPLETED (Evening Session)

### Final video

- Contains: 20-second base loop × 52 (FFmpeg `-stream_loop 51`) + `master_audio.m4a` muxed

### Render pipeline (what actually worked)
1. MoviePy rendered a 20-second seamless base loop (`base_loop.mp4`, 480 frames @ 24fps, ~2.5 min render)
2. FFmpeg looped it 52× and muxed with `master_audio.m4a` via background process
3. FFmpeg command: `-stream_loop 51 -c:v libx264 -b:v 350k -preset fast -c:a aac -b:a 128k -shortest -movflags +faststart`
4. Re-encoding video (not `-c:v copy`) was essential — copy mode produces ~134MB, too big for Vercel

### Key lessons (now in SKILL.md)
- **Always run long FFmpeg as background process** (`&` + redirect to `/tmp/ffmpeg_mux.log`) — OpenClaw exec sessions time out and SIGTERM kills the encode
- **Never `-c:v copy` for final mux** — bitrate stays at source (~1090kbps), output is 134MB. Use `-b:v 350k` for ~61MB at 17:20
- **`-movflags +faststart`** required for browser streaming — file will show as invalid until FFmpeg finishes the moov atom pass at the end
- **Ripple animation seamlessly loops by design** — it's a time-periodic sine formula, no loop-start/end state
- **Dribble every 20s = 20s base loop** — loop duration must be an integer multiple of dribble period
- Use coordinate picker tool at `/coords.html` for any future object positioning — color detection unreliable in illustrated scenes
